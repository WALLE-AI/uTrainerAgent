# 完成前验证技能

在宣布任务完成之前，进行全面验证。

## 核心原则
- **证据优先于声明** - 要有验证结果，不只是说"完成了"
- **多角度验证** - 不只是跑一个测试
- **边界测试** - 正常情况通过不代表完成

## 验证清单

### 1. 功能验证
- [ ] 主要功能是否工作？
- [ ] 边界情况是否处理？
- [ ] 错误情况是否正确处理？

### 2. 回归验证
- [ ] 现有测试是否全部通过？
- [ ] 相关功能是否受影响？
- [ ] 是否有新的警告或错误？

### 3. 集成验证
- [ ] 与其他组件的交互是否正常？
- [ ] API 契约是否保持一致？
- [ ] 数据库迁移是否成功？

### 4. 文档验证
- [ ] README 是否需要更新？
- [ ] API 文档是否需要更新？
- [ ] 变更日志是否更新？

## 验证步骤模板

```markdown
## 验证报告

### 功能测试
- 测试: [描述]
- 结果: [通过/失败]
- 证据: [截图/日志]

### 回归测试
- 命令: `npm test`
- 结果: [X/Y 测试通过]

### 手动验证
- 场景: [描述]
- 步骤: [1, 2, 3...]
- 结果: [描述]
```

## 常见陷阱
- ❌ 只在本地环境测试
- ❌ 只测试正常情况
- ❌ 依赖缓存的旧状态
- ❌ 没有清理测试数据
