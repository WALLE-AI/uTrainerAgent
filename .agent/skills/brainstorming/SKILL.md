# 头脑风暴技能

在开始任何编码工作之前，先进行设计讨论和需求澄清。

## 触发条件
- 用户提出新功能需求
- 开始一个新项目
- 需要重构现有代码

## 工作流程

### 1. 理解需求
- 询问用户真正想要实现什么
- 理解业务背景和限制条件
- 确认成功标准

### 2. 澄清问题
提出关键问题：
- 这个功能的核心价值是什么？
- 有哪些必须满足的约束条件？
- 期望的时间线是什么？
- 有没有类似的参考实现？

### 3. 探索方案
- 提出至少2-3种可行方案
- 分析每种方案的优缺点
- 考虑技术债务和可维护性

### 4. 分段验证
将设计拆分成可消化的小块：
- 每个部分不超过5分钟阅读时间
- 等待用户确认后再继续
- 记录用户的反馈和决策

### 5. 输出设计文档
保存设计决策到 `implementation_plan.md`：
- 问题描述
- 选定方案及理由
- 实施步骤
- 验收标准
